안녕하세요, 잠탱이입니다 👋

드디어 AI/기술 블로그를 오픈했어요. 원래 일상이랑 리뷰는 네이버 블로그로 올리고 있는데, 개발 관련 내용은 따로 정리해두고 싶다는 생각이 계속 있었거든요.

그래서 만들었습니다. **잠탱이의 AI 연구소** 🎉

---

## 왜 블로그를 시작했냐면

개발하다 보면 삽질을 진짜 많이 해요. 에러 해결하는 데만 몇 시간 쓰는 건 기본이고, 겨우 해결하고 나면 "이거 어떻게 풀었더라?" 싶어서 다시 찾아보는 일도 많고요.

그런 내용들을 그냥 흘려보내기가 아까웠어요.

> 내가 삽질한 내용이 누군가한테는 딱 필요한 답이 될 수도 있잖아요.

그래서 시작했습니다. 어렵게 쓰려는 게 아니라, **내가 경험한 걸 솔직하게 기록**하는 공간으로 만들고 싶어요.

---

## 앞으로 어떤 글을 올릴 건지

지금 주로 이런 것들을 공부하고 만들고 있어요:

- **Gemini API** — Google의 LLM API 연동
- **FastAPI** — Python 백엔드 서버
- **LLM 활용** — 챗봇, 자동화, 프롬프트 엔지니어링

그래서 앞으로 올라올 글은 대충 이런 느낌이에요:

| 카테고리 | 예시 |
|---|---|
| 튜토리얼 | Gemini API 발급부터 연동까지 |
| 삽질 후기 | FastAPI + uvicorn 배포하다 만난 에러들 |
| 비교 리뷰 | GPT vs Gemini, 실제로 써보니까 |
| 개발 일지 | AI 챗봇 만들어가는 과정 기록 |
| AI 소식 | 유용한 툴/논문/트렌드 요약 |

비전공자에서 AI 개발자로 전환 중인 과정이라, 전문가 입장보다는 **"나도 해봤는데 이렇더라"** 스타일로 쓸 예정이에요. 어렵게 쓰면 저도 나중에 못 읽거든요 😅

## GitHub Pages로 만든 이유

이 블로그는 **GitHub Pages + Jekyll(Chirpy 테마)** 로 만들었어요.

티스토리도 병행할 예정인데, 이쪽은 좀 더 개발자스럽게 운영하고 싶어서 따로 만들었어요. 마크다운으로 글 쓰고, VS Code에서 push하면 바로 배포되는 게 너무 편하더라고요.

나중에 블로그 세팅 방법도 글로 정리해볼게요.
